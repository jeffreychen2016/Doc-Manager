System Requirements

The application can be installed on the following systems:

  • Windows 7 or newer (32-bit)
  • Windows 7 or newer (64-bit)

Menu

The main menu provides access to additional features of the application.

  • Full Export
  • Check for updates
  • About

Full Export

The full export command resets the export process so that ALL records are exported and uploaded on the next run of the export process. This command applies only to database queries that reference the time of last successful synchronization.

Check for updates

The Check for updates command will connect with the Comdata installation service to check if there are any available updates for this application. You will have the option to download and upgrade your installation if a newer version is available.

About

The About command will display information about the application.

Dashboard

The dashboard is the main screen of the application and provide access to all of the features of the application.

  • Recent Export Status
  • Export History
  • Configuration
  • Export
  • Customer Support

Recent Export Status

The Recent Update Status area of the dashboard displays the status of the most recent export operation.

  • Status displays the result. SUCCESS, ERROR, or RUNNING
  • Start Time displays the time that the export operation started
  • Records displays the total number of records exported and uploaded
  • Errors displays the total number of records that contained errors

NOTE: Only one export operation can be running at a time.

Export History

The Export History area of the dashboard displays a history of export operations. Each entry displays the following:

  • Date displays the time that the export operation started
  • Status displays the result. SUCCESS or ERROR

Click on an entry in this list to view detailed results of the export operation.

Export History Details

The Export History Details displays detailed results of the selected export operation.

  • Export Status
    • Status displays the result. SUCCESS or ERROR
    • Start Time displays the time that the export operation started
    • End time displays the time that the export opertion completed
    • Records displays the total number of records exported and uploaded
    • Errors displays the total number of records that contained errors
    • Accounting System displays the accounting system used during the export
    • Database server displays the database server used during the export
  • Details
    • Details for each database and each table exported for each database are listed in a hierarchical view. Each entry displays
      • Total number of records successfully exported
      • Total number of records attempted
      • Total number of records containing errors
      • Total number of bytes in all of the records exported

Configuration

Clicking on the Configuration button on the Dashboard displays the configuration panel used to set all of the options for the export process.

  • Apply: This button will save any changes made to the configuration settings
  • Save: This button will save any changes made to the configuration settings and return to the Dashboard
  • Close: This button will close the configuration panel and return to the Dashboard
  • Cancel: This button will cancel any unsaved changes made to the configuration settings and return to the Dashboard

The configuration panel contains multiple tabs used to define the settings for the export process:

  • Account Information
  • Database Connections
  • Data Export
  • Custom Properties

Account Information

The Account Information tab is used to define the settings for the tenant identifier, PIN, and accounting system to use in the export process:

  • Tenant ID: Enter your tenant ID
  • Tenant PIN: Enter the tenant PIN (aka “Password”)
  • Add: Click Add button to add the tenant to the list.
  • Remove: Select a tenant from the list and click Remove button to remove the tenant from the list
  • Accounting System: Select the accounting system / version for your system. This list is enabled when accounting system configuration files have been downloaded.
  • Download Updates: Click this button to connect with the CEMS service to download the available accounting system configuration files for your tenant ID. This button is enabled when a tenant is selected from the tenant list.

NOTE: The Accounting System selections are initially empty. You must enter your Tenant ID and Tenant PIN and then click on the Download Updates button to retrieve the available accounting system configuration files for your company ID. Once the download completes, the Accounting System selections will be populated with the available selections.

You can click on the Download Updates button any time that you need to check for updated accounting system configuration files.

Database Connections

The Database Connections tab is used to define the databases to use in the export process:

  • Tenant: This dropdown will display all the tenants added in the account information configuration page. First tenant will be selected by default. Select the desired tenant from the list.
  • Database Server: Enter your database server address (if applicable)
  • Database Name or DSN Name: Enter the database or DSN name
  • Add: Click this button to add the database to the list of defined databases. This button is enabled when a name is entered for the database.
  • Remove: Click this button to remove the selected database from the list of defined databases. This button is enabled when a database is selected in the list of defined databases.
  • Username: Enter the username for authentication (if applicable)
  • Password: Enter the password for authentication (if applicable)
  • Test Connection: Click this button to verify the connection settings for the selected database. This button is enabled when a database is selected in the list of defined databases.

Data Export

The Data Export tab is used to select the coding values used in the export process:

  • Tenant: This dropdown will display all the tenants added in the account information configuration page. First tenant will be selected by default. Select the desired tenant from the list.
  • Database: This dropdown will display the databases added for the selected tenant. Select the database to use for selecting the coding values
  • Select your data: Select the coding type to use for selecting the coding value
  • Select All: Click this button to select all coding values in the list
  • Unselect All: Click this button to unselect all coding values in the list
  • Coding values: Check / select each of the coding values to export for the current selected coding type.

The available databases are defined in the Database Connections tab.

The coding types are defined in the accounting system configuration file that is selected on the Account Information tab.

The coding values that you select on this tab are used to filter the data during the export process.

NOTE that the selected coding values are associated with the selected database. If you have multiple databases defined then you must select coding values for each coding type for each database.

Custom Properties

The Custom Properties tab is used to define any additional application properties:

  • Edit: Click this button to modify the value for the selected property. This button is enabled when a property is selected.
  • Reset: Click this button to reset the value for the selected property to its default value. This button is enabled when a property is selected.
  • Reset All Properties: Click this button to reset all of the listed custom properties to their default values.

Application-wide properties have a ‘app.’ prefix. Please consult with your Comdata customer support representative before changing any of these properties.

Export

Clicking on the Export button on the Dashboard starts the export operation. The export process will export data for all the configured tenants sequentially in a single run. When the export operation is running, the Status field of the Recent Export Status display is set to RUNNING and will flash until completed. When the export operation completes, the Status field of the Recent Export Status display is set to either SUCCESS or ERROR depending on the result.

You can view the details of all completed export operations by clicking on one of the entries in the Export History display.

NOTE: Only one export operation can be running at a time.

Customer Support

Clicking on the Customer Support button on the Dashboard displays the customer support panel where you can collect application information to send to your customer support representative. This information will assist in resolving any issues that you may be having with the export tool.

  • Application Configuration File: The current configuration settings for the application.
  • Accounting System Configuration Files: The set of accounting system configuration files downloaded on the Account Information configuration tab.
  • Application Log File: The application log file.
  • Export Log File: The log file from the most recent export operations.
  • Export History Files: The analytics result files from the most recent export operations.
  • System Information: Information about your computer system, system components, memory, disk space, etc.

After clicking the Save button, a zip file will be created containing the selected information. You should send that zip file to your customer support representative to assist in resolving any issues.

Notifications (Windows)

The application generates the following system application event notifications:

Comdata Export Sync Tool

Upon completion of each export operation, an event is created in the Windows application log.

NOTE: The event is only created when the export process is running with elevated privileges.

The possible values contained in the event include:

  • Source: Comdata Export Sync Tool
  • Level: Information
  • Event ID:
    • 1 = successful
    • 999 = error
  • Description: Contains the result and a reference to the analytics file

Logging

Log files are located in Logs folder of the application data directory. There are three different log files generated by the application.

Application Log

The application log contains information about the activity of the application. This file is named ExportSyncTool-Client.log and is located in the Logs folder of the application data directory (e.g. C:\ProgramData\Comdata\ExportSyncTool\Logs).

Export Log

The export log contains information about the activity of the export operations. This file is named ExportSyncTool-Export.log and is located in the Logs folder of the application data directory (e.g. C:\ProgramData\Comdata\ExportSyncTool\Logs).

Analytics Report

The analytics report contains detailed analytics about each export operation. A separate file is generated for each export operation and is located in the **Logs* folder of the application data directory (e.g. C:\ProgramData\Comdata\ExportSyncTool\Logs).

Software Updates

Updates to the application may occur from time to time. To check for updates to the current application installation, select Check for updates from the application menu.

Scheduled Tasks (Windows)

The export operation can be scheduled to run unattended at specific intervals. On Windows systems, this can be done using the Windows Task Scheduler.

Follow these steps to create the scheduled task:

NOTE: The screens are provided for illustration purposes. Your screens may appear differently depending on your version of Windows.

  • Open Windows Task Scheduler
  • Click Create Task

  • Enter the name for the task and select the security options. You should check Run with highest privileges if you require the Windows event notifications to monitor the export process.

  • Select the Triggers tab

  • Click New to create a new trigger
  • Set the trigger properties and click OK
  • Select the Actions tab
  • Click Browse... and select the location of the ExportSyncTool.exe application.
  • Enter -silent -console in the Add arguments text field.
  • Enter the folder containing the ExportSyncTool.exe application in the Start in field.
  • Click OK
  • Enter any additional settings in the Conditions and Settings tabs.
  • Click OK to complete the creation of the scheduled task.
